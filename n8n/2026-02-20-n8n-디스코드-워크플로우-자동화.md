### Context
n8n을 사용하여 Discord에서 메시지를 수신하면 자동으로 GitHub에 TIL 형식의 코드를 푸시하고, 최종적으로 메시지 전송이 완료되도록 트리거 하는 워크플로우를 만들었습니다. 이 작업은 개발자가 반복적으로 수행해야 하는 작업을 자동화하여 생산성을 높이는 데 목적이 있습니다.

### Core
1. **트리거 노드 설정**: Discord에서 메시지를 수신할 때 트리거 됩니다.
2. **에이전트 실행**: 메시지를 TIL 형식으로 변환합니다.
3. **GitHub 푸시**: 변환된 내용을 코드로 GitHub에 푸시합니다.
4. **README.md 업데이트**: 새로운 인덱스를 파일에 추가합니다.
5. **완료 메시지 발신**: 모든 작업이 완료되면 확인 메시지를 보냅니다.

```json
{
  "discord": {
    "trigger": "New Message In Channel",
    "output": "TIL formatted content"
  },
  "github": {
    "actions": ["commit","push"],
    "update": "README.md"
  }
}
```

### Insight
n8n을 처음 사용해보았는데, 노드의 입력(input)과 출력(output)만 정확히 이해하면 워크플로우를 구성하는 것이 어렵지 않았습니다. 단순한 개념으로도 복잡한 자동화를 구현할 수 있었습니다. 이는 n8n의 강력한 기능을 잘 활용할 수 있는 기회였습니다.
